# TypeScript 5.6 Breaking Changes & Migration

Date: 2026-04-21
Tag: [typescript] [v5.6] [breaking-changes] [migration]

## Overview

TypeScript 5.6 adds iterator helper methods, new compiler options for build errors and side effect imports, and improved type checking for nullish/truthy expressions.

## New Features

### 1. Disallowed Nullish and Truthy Checks

The compiler now errors when it can syntactically determine a truthy or nullish check will always evaluate in a specific way.

Catches bugs like:
- Forgetting `.test()` on regex: `/123/.test(input)` → `/123/.test(input)` (not typo)
- Using `=>` instead of `>=`

### 2. Iterator Helper Methods

New methods on iterators produced by generators and methods like `entries()` on Maps/Sets:

```typescript
// Example
const map = new Map([["a", 1], ["b", 2]]);
const mapped = map.entries().map(([k, v]) => [k, v * 2]);
```

Available methods: `map`, `filter`, `take`, `takeWhile`, `drop`, `dropWhile`, `flatMap`, `reduce`

### 3. New Types

- `IteratorObject` (renamed from `BuiltinIterator`)
- Subtypes: `ArrayIterator`, `MapIterator`, `SetIterator`
- `AsyncIteratorObject` for async parity
- `BuiltinIteratorReturn` intrinsic type

### 4. New Compiler Options

| Option | Description |
|--------|-------------|
| `--stopOnBuildErrors` | Stops build on first project with errors |
| `--noUncheckedSideEffectImports` | Errors if side effect imports can't be resolved |
| `--noCheck` | Skips type checking for faster emit |
| `--strictBuiltinIteratorReturn` | Ensures `BuiltinIteratorReturn` is `undefined` instead of `any` |

### 5. Support for Arbitrary Module Identifiers

Allows exports/imports with invalid identifier names as string literals:

```typescript
export { banana as "🍌" };
import { "🍌" as banana } from "emoji";
```

### 6. Editor Improvements

- Region-prioritized diagnostics for faster feedback in large files
- Granular commit characters for better auto-complete behavior
- `autoImportSpecifierExcludeRegexes` setting for filtering auto-import suggestions

## Breaking Changes

### 1. `.tsbuildinfo` Always Written

Now emitted for any project in `--build` invocation regardless of `--incremental` setting.

### 2. File Extension and `package.json` Respect in `node_modules`

Module format info (.mjs, .mts, .cts, package.json "type" field) now respected in all module modes, not just `node16`/`nodenext`.

### 3. Correct `override` Checks on Computed Properties

Previously, computed properties marked with `override` did not correctly verify base class member existence.

## Migration Notes

`--noUncheckedSideEffectImports` can be enabled gradually - working code importing non-existent modules may need ambient module declarations or path corrections.

## Impact Assessment

| Area | Impact |
|------|--------|
| Iterator helpers | Positive |
| Nullish/truthy checks | Positive (catches bugs) |
| .tsbuildinfo | Low |
| Module resolution | Low |
