# TypeScript 5.4 Breaking Changes & Migration

Date: 2026-04-21
Tag: [typescript] [v5.4] [breaking-changes] [migration]

## Overview

TypeScript 5.4 preserves narrowing in closures following last assignments, introduces `NoInfer` utility type, and adds support for `Object.groupBy` and `Map.groupBy`.

## New Features

### 1. Preserved Narrowing in Closures Following Last Assignments

TypeScript 5.4 now tracks the last assignment point for parameters and `let` variables in non-hoisted functions, preserving narrowing from outside the containing function:

```typescript
function example() {
  let value: string | null = "hello";

  // In 5.4, x is narrowed to string inside the callback
  const callback = () => {
    console.log(value.toUpperCase()); // No error
  };

  value = null; // Last assignment tracked
  callback();
}
```

### 2. The `NoInfer` Utility Type

Surrounding a type in `NoInfer<...>` signals TypeScript not to dig in and match against inner types:

```typescript
function createStreetLight<C extends string>(
  colors: C[],
  defaultColor?: NoInfer<C>
) { /* ... */ }

// Here, defaultColor won't be inferred from colors
```

### 3. `Object.groupBy` and `Map.groupBy`

```typescript
// Object version
const people = [
  { name: "Alice", age: 30 },
  { name: "Bob", age: 25 },
  { name: "Carol", age: 30 }
];

const grouped = Object.groupBy(people, person => person.age);
// { "25": [{ name: "Bob", age: 25 }], "30": [...] }

// Map version
const mapGrouped = Map.groupBy(people, person => person.age);
```

Requires `--target esnext` or adjusted `lib` settings.

### 4. Support for `require()` calls with `--module preserve`

New `--module preserve` option allows `require()` calls with `--moduleResolution bundler`, accurately modeling modern bundler behavior.

### 5. Checked Import Attributes

Import attributes are now checked against the global `ImportAttributes` type.

### 6. Quick Fix for Adding Missing Parameters

TypeScript now offers a quick fix to add new parameters to functions called with too many arguments.

### 7. Auto-Import Support for Subpath Imports

TypeScript's auto-imports now support Node.js subpath imports via the `imports` field in `package.json`.

## Notable Behavioral Changes

### 1. More Accurate Conditional Type Constraints

Conditional types with generic parameters may no longer be compatible with their branches.

### 2. Aggressive Intersection Reduction

Intersections between type variables and primitives are now reduced more aggressively:

```typescript
// T & string with T extends "abc" | "def" becomes just T
```

### 3. Template String Checking

More accurate assignment checking for template strings with interpolations.

### 4. Type-Only Import Conflicts

Imports conflicting with local values now error under `isolatedModules`.

### 5. Enum Assignability

Enums with same names but different values are no longer compatible.

### 6. Enum Member Naming

Enum members can no longer use `Infinity`, `-Infinity`, or `NaN` as names.

### 7. Mapped Types with `any` Rest Elements

Tuples with `any` rest elements now preserve correct types instead of becoming `any`.

## Migration Notes

Most changes are additive or bug fixes. The enum member naming restriction may affect existing code.
