# TypeScript 6.0 Breaking Changes & Migration

Date: 2026-04-21
Tag: [typescript] [v6] [breaking-changes] [migration]

## Overview

TypeScript 6.0 is a major version with significant default changes and deprecations. Many compiler options that were previously optional are now changed or removed.

## Breaking Changes

### Compiler Defaults Changed

| Option | Old Default | New Default |
|--------|-------------|-------------|
| `strict` | `false` | `true` |
| `module` | `commonjs` | `esnext` |
| `target` | `ES3` | Current year ES version |
| `noUncheckedSideEffectImports` | `false` | `true` |
| `libReplacement` | `true` | `false` |

### Removed Options

The following options are completely removed (use alternatives):

| Removed | Alternative |
|---------|------------|
| `moduleResolution: node` | `bundler`, `node16`, or `nodenext` |
| `moduleResolution: classic` | `bundler` |
| `rootDir` (default behavior) | Explicitly set if needed |
| `types` default (all @types) | `[]` (explicit type list required) |
| `target: es5` | Minimum is `es2015` |
| `baseUrl` | Use subpath imports |
| `esModuleInterop: false` | Always enabled |
| `allowSyntheticDefaultImports: false` | Always enabled |
| `alwaysStrict: false` | Always enabled |
| `--downlevelIteration` | Removed (always on) |
| `module: amd`, `umd`, `systemjs`, `none` | Use `esnext`/`commonjs` |
| `outFile` | Use bundler |
| `assert` keyword on imports | Use `with` |
| `no-default-lib` directive | Removed |
| Command-line files with existing `tsconfig.json` | Error |

### Key Behavioral Changes

1. **`types: []` by default** - No `@types/*` packages auto-included. Must explicitly list types:
   ```json
   {
     "compilerOptions": {
       "types": ["node"]
     }
   }
   ```

2. **`rootDir` defaults to project root** - May expose files not meant to be compiled

3. **Subpath Imports `#/`** - Packages can use `#/*` prefix for subpath imports under `nodenext`/`bundler`

4. **`es2025` Target/Lib** - New target with `RegExp.escape` and staged features

5. **Temporal API Types** - Built-in types for ECMAScript Temporal proposal

6. **Map Upsert Methods** - `getOrInsert` and `getOrInsertComputed` on Map/WeakMap

7. **Consolidated DOM Libs** - `dom.iterable` and `dom.asynciterable` merged into `lib.dom.d.ts`

8. **`--stableTypeOrdering` Flag** - Matches TypeScript 7.0 behavior for smoother future migration

9. **Less Context-Sensitivity on `this`-less Functions** - Better inference for functions not using `this`

## Migration Guide

### Step 1: Update tsconfig.json

```json
{
  "compilerOptions": {
    "types": ["node"],
    "rootDir": "./src"
  }
}
```

### Step 2: Handle Strict Mode

If you weren't using `strict: true`, review:
- `strictNullChecks`
- `noImplicitAny`
- `strictFunctionTypes`
- etc.

### Step 3: Fix Import Attributes

```typescript
// Old (deprecated)
import foo from "foo" assert { type: "json" }

// New
import foo from "foo" with { type: "json" }
```

### Step 4: Update @types packages

```bash
npm update @types/node --save-dev
```

### Step 5: Use `ignoreDeprecations` for gradual migration

```json
{
  "compilerOptions": {
    "ignoreDeprecations": "6.0"
  }
}
```

### Step 6: Automated Tool (Experimental)

TypeScript provides an experimental migration tool:
```bash
npx ts5to6
```

This can automate some adjustments.
