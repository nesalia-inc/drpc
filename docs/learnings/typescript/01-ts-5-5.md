# TypeScript 5.5 Breaking Changes & Migration

Date: 2026-04-21
Tag: [typescript] [v5.5] [breaking-changes] [migration]

## Overview

TypeScript 5.5 introduces inferred type predicates, JSDoc `@import` tag, regex syntax checking, and the `isolatedDeclarations` flag. Also formally deprecates several older options.

## New Features

### 1. Inferred Type Predicates

TypeScript now infers type predicates from functions with specific conditions:

```typescript
// Before 5.5: inferred as (string | null)[]
// After 5.5: inferred as string[]
const strings = values.filter(x => x !== null);
```

This enables better type narrowing in filter operations.

### 2. Control Flow Narrowing for Constant Indexed Accesses

`obj[key]` expressions are now narrowed when both are effectively constant.

### 3. JSDoc `@import` Tag

New syntax for type-only imports in JSDoc comments:

```typescript
/**
 * @import { SomeType } from "module"
 */
```

### 4. Regular Expression Syntax Checking

Basic syntax checking on regex literals, catching:
- Backreference errors
- Unavailable features per target ES version

### 5. New ECMAScript `Set` Methods

```typescript
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);

a.union(b);           // Set { 1, 2, 3, 4 }
a.intersection(b);    // Set { 2, 3 }
a.difference(b);      // Set { 1 }
a.symmetricDifference(b); // Set { 1, 4 }
a.isSubsetOf(b);      // false
a.isSupersetOf(b);   // false
a.isDisjointFrom(b);  // false
```

### 6. Isolated Declarations

New `--isolatedDeclarations` flag requiring explicit type annotations on exports for parallel declaration emit:

```typescript
// Requires explicit type annotation
export function foo(x: string): string { return x; }
```

### 7. `${configDir}` Template Variable

Resolves paths relative to the derived config file, not the extended base.

### 8. `transpileDeclaration` API

New API for generating single declaration files without full program context.

### 9. Performance Improvements

- 5-8% build time improvement
- 10-20% faster language service operations
- 2x speedup with `transpileModule`

## Breaking Changes

### 1. Decorators Stricter Parsing

Stricter parsing may require parenthesization:

```typescript
// Old
@super.decorate

// New
@(super.decorate)
```

### 2. `undefined` as Type Name

```typescript
// Now an error
type undefined = any;
```

### 3. Reference Directive Emit

Directives no longer synthesized or preserved unless `preserve="true"` is added.

## Deprecated Options (Now Disabled)

TypeScript 5.5 hard-errors on previously deprecated options. If you were using `ignoreDeprecations: "5.0"`, you must update:

| Option | Status |
|--------|--------|
| `charset` | Removed |
| `target: ES3` | Removed |
| `importsNotUsedAsValues` | Removed |
| `noImplicitUseStrict` | Removed |
| `noStrictGenericChecks` | Removed |
| `keyofStringsOnly` | Removed |
| `suppressExcessPropertyErrors` | Removed |
| `suppressImplicitAnyIndexErrors` | Removed |
| `out` | Removed |
| `preserveValueImports` | Removed |
| `prepend` in project references | Removed |
| OS-specific `newLine` | Removed |

## Migration Notes

### Type Predicate Changes

Filter callbacks that previously inferred `boolean` may now infer type predicates:

```typescript
// Explicit annotation to preserve previous behavior
const nums: (number | null)[] = [1, null].filter(x => x !== null);
```

### Fix Decorator Errors

```typescript
// If you have @super.decorate, change to:
@(super.decorate)
class MyClass {}
```
