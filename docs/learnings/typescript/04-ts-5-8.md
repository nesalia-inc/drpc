# TypeScript 5.8 Breaking Changes & Migration

Date: 2026-04-21
Tag: [typescript] [v5.8] [breaking-changes] [migration]

## Overview

TypeScript 5.8 focuses on ESM compatibility, new compiler flags, and stabilizing features from earlier versions.

## New Features

### 1. Granular Checks for Return Branches

TypeScript now checks each branch of conditional expressions in return statements against the declared return type.

### 2. `require()` ESM Support

TypeScript 5.8 supports `require("esm")` calls from CommonJS to ECMAScript modules under `--module nodenext` flag.

### 3. Stable `--module node18` Flag

New stable module target for Node.js 18 users.

### 4. `--erasableSyntaxOnly` Option

Flags TypeScript constructs with runtime semantics:
- Enums
- Namespaces with runtime code
- Parameter properties
- `import =` aliases

### 5. `--libReplacement` Flag

Allows disabling automatic lib file substitution behavior.

### 6. Computed Property Names in Declarations

Preserves computed property names in emitted declaration files.

### 7. Performance Optimizations

- Improved path normalization
- Avoids re-validating options on structural edits

## Breaking Changes

### 1. `lib.d.ts` Updates

DOM type changes may introduce new type-checking errors.

### 2. Import Assertions Deprecated

Under `--module nodenext`, import assertions using `assert` syntax now error. Use import attributes with `with` keyword instead:

```typescript
// Old (deprecated)
import foo from "foo" assert { type: "json" }

// New
import foo from "foo" with { type: "json" }
```

## Migration

Minimal migration effort - primarily additive features. The `assert` → `with` migration can be done gradually.

## Impact Assessment

| Area | Impact |
|------|--------|
| Import assertions | Low |
| DOM types | Low |
