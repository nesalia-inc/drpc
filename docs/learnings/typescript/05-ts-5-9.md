# TypeScript 5.9 Breaking Changes & Migration

Date: 2026-04-21
Tag: [typescript] [v5.9] [breaking-changes] [migration]

## Overview

TypeScript 5.9 introduces a more prescriptive `tsc --init` template, support for `import defer`, and important `lib.d.ts` changes affecting TypedArray types.

## New Features

### 1. Minimal and Updated `tsc --init`

The generated `tsconfig.json` is now more prescriptive with sensible defaults:

```json
{
  "compilerOptions": {
    "module": "nodenext",
    "target": "esnext",
    "strict": true,
    "jsx": "react-jsx",
    "moduleDetection": "force"
  }
}
```

### 2. Support for `import defer`

TypeScript 5.9 supports ECMAScript's deferred module evaluation proposal:

```typescript
import defer * as feature from "./some-feature.js"
```

The module only executes when an export is first accessed.

### 3. Support for `--module node20`

A stable option modeling Node.js v20 behavior. Implies `--target es2023` unless configured.

### 4. Expandable Hovers (Preview)

New "quick info verbosity" feature with +/- buttons on hover tooltips.

### 5. Configurable Maximum Hover Length

New `js/ts.hover.maximumLength` setting in VS Code.

### 6. Performance Optimizations

- **Cache Instantiations on Mappers**: Caches intermediate type instantiations (significant for Zod, tRPC)
- **Avoiding Closure Creation**: ~11% speed-up in file existence checks

## Breaking Changes

### 1. `lib.d.ts` TypedArray Changes (HIGH IMPACT)

`ArrayBuffer` is no longer a supertype of TypedArray types including Node.js `Buffer`.

Common errors:
```
"Argument of type 'ArrayBufferLike' is not assignable to parameter of type 'BufferSource'"
"Type 'Buffer' is not assignable to type 'Uint8Array<ArrayBufferLike>'"
```

### 2. Type Argument Inference Changes

Fixes for type variable "leaks" during inference may introduce new type errors in some codebases.

## Migration Notes

### Fix TypedArray/Buffer Issues

1. **Update @types/node**:
   ```bash
   npm update @types/node --save-dev
   ```

2. **Use explicit buffer types**:
   ```typescript
   // Instead of:
   someFunc(data) // data is Uint8Array

   // Use:
   someFunc(data.buffer as ArrayBuffer)
   // or
   someFunc(new Uint8Array(data) as Uint8Array<ArrayBuffer>)
   ```

3. **Explicit type arguments**: If encountering new errors with generic functions, add explicit type parameters.

## Impact Assessment

| Area | Impact |
|------|--------|
| TypedArray/Buffer types | High |
| Type inference | Medium |
| Performance | Positive |
