Next.js is the primary target for DeesseJS RPC. The `@deessejs/server-next` package provides seamless integration with the Next.js App Router.

## Requirements

- Next.js 14+ (App Router)
- TypeScript 5.0+
- `@deessejs/server` installed

## Installation

```bash
npm install @deessejs/server-next
# or
pnpm add @deessejs/server-next
```

## Quick Setup

### 1. Define Your API

Create your API definition in a separate file:

```typescript
// src/server/api.ts
import { defineContext, createPublicAPI } from '@deessejs/server'
import { z } from 'zod'
import { ok, err } from '@deessejs/fp'

const { t } = defineContext({
  context: () => ({ db: myDatabase }),
})

// Public query - callable from client
const getUser = t.query({
  args: z.object({ id: z.number() }),
  handler: async (ctx, args) => {
    const user = await ctx.db.users.find(args.id)
    if (!user) return err({ code: 'NOT_FOUND', message: 'User not found' })
    return ok(user)
  },
})

// Public mutation - callable from client
const createUser = t.mutation({
  args: z.object({
    name: z.string(),
    email: z.string().email(),
  }),
  handler: async (ctx, args) => {
    const user = await ctx.db.users.create(args)
    return ok(user, { invalidate: ['users:list'] })
  },
})

// Internal query - server only, NOT exposed via HTTP
const getAdminStats = t.internalQuery({
  handler: async (ctx) => {
    return ok({ totalUsers: await ctx.db.users.count() })
  },
})

// Export router with type
export const appRouter = t.router({
  users: t.router({
    get: getUser,
    create: createUser,
    getAdminStats,
  }),
})

// Create client-safe API (excludes internal operations)
export const clientRouter = createPublicAPI(appRouter)

export type AppRouter = typeof appRouter
```

### 2. Create the Route Handler

Create a catch-all route that handles all RPC requests:

```typescript
// app/api/[...route]/route.ts
import { createNextHandler } from '@deessejs/server-next'
import { clientRouter } from '@/server/api'

export const { GET, POST, PUT, PATCH, DELETE } = createNextHandler(clientRouter)
```

This creates endpoints like `POST /api/users.get` and `POST /api/users.create`.

### 3. Use on the Server

You can call operations directly from Server Components or Server Actions:

```typescript
// app/admin/page.tsx
import { appRouter } from '@/server/api'

export default async function AdminPage() {
  // Works - internal operations available on server
  const stats = await appRouter.users.getAdminStats({})

  return (
    <div>
      <h1>Admin Dashboard</h1>
      <p>Total users: {stats.value.totalUsers}</p>
    </div>
  )
}
```

### 4. Use on the Client

Create a typed client and use it in your components:

```typescript
// src/lib/client.ts
import { createClient } from '@deessejs/client-react'
import { httpBatchLink } from '@deessejs/client'
import type { AppRouter } from '@/server/api'

export const client = createClient<AppRouter>({
  links: [httpBatchLink({ url: '/api' })],
})
```

```typescript
// app/components/UserList.tsx
'use client'

import { client } from '@/lib/client'

async function UserProfile({ id }: { id: number }) {
  const result = await client.users.get.query({ id })

  if (result.ok) {
    return <div>{result.value.name}</div>
  }

  return <div>User not found</div>
}
```

## Project Structure

Your project should look like this:

```
my-app/
├── app/
│   ├── api/
│   │   └── [...route]/
│   │       └── route.ts    # DeesseJS RPC handler
│   ├── admin/
│   │   └── page.tsx        # Server Component (uses internal ops)
│   └── components/
│       └── UserList.tsx    # Client Component (uses public ops)
├── lib/
│   └── client.ts           # Client configuration
└── server/
    └── api.ts              # API definitions
```

## With Authentication

Combine DeesseJS RPC with authentication libraries like better-auth:

```typescript
// app/api/[...route]/route.ts
import { createNextHandler } from '@deessejs/server-next'
import { auth } from '@/lib/auth'
import { toNextJsHandler } from 'better-auth/next-js'
import { clientRouter } from '@/server/api'

// DeesseJS RPC handler
const drpc = createNextHandler(clientRouter)

// better-auth handler
const { GET, POST } = toNextJsHandler(auth)

export { GET, POST, drpc }
```

## How It Works

The Next.js adapter uses a catch-all route `[...route]` to handle RPC calls:

1. **HTTP Method** - Uses appropriate method (GET for queries, POST for mutations)
2. **Route Pattern** - `POST /api/users.get` calls the `get` operation in the `users` router
3. **Request Body** - Contains `{ args: {...} }` for the operation arguments
4. **Response Format** - Always returns `{ ok: true, value: {...} }` or `{ ok: false, error: {...} }`

<Callout type="info">
Internal operations (`internalQuery`, `internalMutation`) are NOT exposed via HTTP. They can only be called from server-side code.
</Callout>

## Common Patterns

### Calling from Server Actions

```typescript
// app/actions.ts
'use server'

import { appRouter } from '@/server/api'

export async function createUserAction(data: { name: string; email: string }) {
  const result = await appRouter.users.create({
    name: data.name,
    email: data.email,
  })

  if (!result.ok) {
    throw new Error(result.error.message)
  }

  return result.value
}
```

### Using in Client Components

```typescript
// app/components/CreateUser.tsx
'use client'

import { client } from '@/lib/client'
import { useState } from 'react'

export function CreateUser() {
  const [name, setName] = useState('')
  const [email, setEmail] = useState('')

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault()
    const result = await client.users.create.mutation({ name, email })

    if (result.ok) {
      console.log('Created:', result.value)
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={name} onChange={(e) => setName(e.target.value)} />
      <input value={email} onChange={(e) => setEmail(e.target.value)} />
      <button type="submit">Create</button>
    </form>
  )
}
```

## Anti-Patterns

### Don't Expose Internal Operations to Clients

```typescript
// ❌ Don't - exposes internal operations
export const { GET, POST } = createNextHandler({
  router: appRouter, // Contains internal operations!
})
```

```typescript
// ✅ Do - use createPublicAPI to filter internal operations
export const { GET, POST } = createNextHandler(clientRouter)
```

### Don't Use Internal Operations from Client Components

```typescript
// ❌ Don't - will fail because it's not exposed via HTTP
const stats = await client.users.getAdminStats.query({})
```

```typescript
// ✅ Do - call internal operations from server code only
const stats = await appRouter.users.getAdminStats({})
```
