## Introduction

DeesseJS RPC is a type-safe RPC protocol for Node.js and browsers with full TypeScript inference. Define your API once, call it from anywhere with full type safety.

## Terminology

| Term | Description |
|------|-------------|
| **Query** | A public read operation, callable from client and server |
| **Mutation** | A public write operation, callable from client and server |
| **Internal Query** | A server-only read operation, not exposed via HTTP |
| **Internal Mutation** | A server-only write operation, not exposed via HTTP |
| **Context** | The shared state passed to all procedure handlers |
| **Router** | A collection of procedures grouped under a namespace |

## Requirements

- TypeScript 5.0+
- Node.js 20+

```bash
npm install @deessejs/server
```

## Create Your First API

<Steps>
  <Step>
    ### Define Context

    Start by defining your context with the services you need:

    ```typescript
    // src/api.ts
    import { defineContext } from '@deessejs/server'
    import { z } from 'zod'

    const { t, createAPI } = defineContext({
      context: () => ({ db: myDatabase }),
    })
    ```
  </Step>
  <Step>
    ### Define a Query

    Create a typed query that fetches data and returns cache keys:

    ```typescript
    // src/api.ts
    import { ok } from '@deessejs/fp'

    const getUser = t.query({
      args: z.object({ id: z.number() }),
      handler: async (ctx, args) => {
        const user = await ctx.db.users.find(args.id)
        return ok(user, { keys: [['users', { id: args.id }]] })
      },
    })
    ```
  </Step>
  <Step>
    ### Define a Mutation

    Create a mutation that modifies data and invalidates cache:

    ```typescript
    // src/api.ts
    const createUser = t.mutation({
      args: z.object({ name: z.string(), email: z.string().email() }),
      handler: async (ctx, args) => {
        const user = await ctx.db.users.create(args)
        return ok(user, { invalidate: ['users:list'] })
      },
    })
    ```
  </Step>
  <Step>
    ### Export the Router

    Combine everything into a router and export the type:

    ```typescript
    // src/api.ts
    export const appRouter = t.router({
      users: t.router({
        get: getUser,
        create: createUser,
      }),
    })

    export type AppRouter = typeof appRouter
    ```

    <Callout type="info" title="Why export the type?">
    The `AppRouter` type is used by clients to get full TypeScript inference. Import it in your client code.
    </Callout>
  </Step>
</Steps>

## Setup Server

Choose your framework and install the adapter:

<Tabs items={['Next.js App Router', 'Hono']} defaultIndex={0}>
  <Tab value="Next.js App Router">
    **File:** `app/api/[...route]/route.ts`

    ```typescript
    // app/api/[...route]/route.ts
    import { createServerHandler } from '@deessejs/server-next'
    import { appRouter } from '@/api'

    export const { api: serverApi, startServer } = createServerHandler({
      router: appRouter,
    })
    ```

    Start the development server with `npm run dev`.
  </Tab>
  <Tab value="Hono">
    **File:** `src/index.ts`

    ```typescript
    // src/index.ts
    import { createHonoHandler } from '@deessejs/server-hono'
    import { Hono } from 'hono'
    import { appRouter } from './api'

    const app = new Hono()
    app.route('/api', createHonoHandler({ router: appRouter }))
    ```

    Start with `bun dev` or `npm run dev`.
  </Tab>
</Tabs>

<Callout type="warn" title="Security">
Only `query` and `mutation` are exposed via HTTP. Use `internalQuery` and `internalMutation` for server-only operations that should never be callable from clients.
</Callout>

## Setup Client

Choose your client library:

<Tabs items={['React', 'Vanilla JS']} defaultIndex={0}>
  <Tab value="React">
    **File:** `src/lib/client.ts`

    ```typescript
    // src/lib/client.ts
    import { createClient } from '@deessejs/client-react'
    import { httpBatchLink } from '@deessejs/client'
    import type { AppRouter } from '@/api'

    export const client = createClient<AppRouter>({
      links: [httpBatchLink({ url: '/api' })],
    })
    ```

    **Usage in a component:**

    ```typescript
    // src/components/user-list.tsx
    import { client } from '@/lib/client'

    // Query
    const user = await client.users.get.query({ id: 1 })
    if (user.ok) {
      console.log(user.value) // fully typed
    }

    // Mutation
    const result = await client.users.create.mutation({
      name: 'Jane',
      email: 'jane@example.com',
    })
    ```
  </Tab>
  <Tab value="Vanilla JS">
    **File:** `src/client.ts`

    ```typescript
    // src/client.ts
    import { createClient } from '@deessejs/client'
    import type { AppRouter } from './api'

    export const client = createClient<AppRouter>({ url: '/api' })
    ```

    **Usage:**

    ```typescript
    // src/client.ts
    import { client } from './client'

    // Query
    const user = await client.users.get({ id: 1 })

    // Mutation
    await client.users.create({ name: 'Jane', email: 'jane@example.com' })
    ```
  </Tab>
</Tabs>

## FAQ

<FaqAccordion items={[
  {
    question: "What's the difference between queries and mutations?",
    answer: "Queries are read operations that can be cached. Mutations are write operations that trigger cache invalidation. Both are exposed via HTTP and callable from clients."
  },
  {
    question: "How does the cache system work?",
    answer: "Procedures return cache keys with results using the `ok()` helper. The cache system automatically invalidates related keys when mutations are called. Use `{ keys: [...] }` for queries and `{ invalidate: [...] }` for mutations."
  },
  {
    question: "What's the difference between public and internal procedures?",
    answer: "Use `t.query` and `t.mutation` for client-callable procedures. Use `t.internalQuery` and `t.internalMutation` for server-only operations that should not be exposed via HTTP."
  },
  {
    question: "Is DeesseJS RPC framework-agnostic?",
    answer: "Yes. @deessejs/server is core and framework-agnostic. Official adapters exist for Next.js, Hono, and Electron."
  }
]} />
