## Introduction

DeesseJS RPC is a type-safe RPC protocol for Node.js and browsers with full TypeScript inference. Define your API once, call it from anywhere with full type safety.

## Terminology

| Term | Description |
|------|-------------|
| **Query** | A public read operation, callable from client and server |
| **Mutation** | A public write operation, callable from client and server |
| **Internal Query** | A server-only read operation, not exposed via HTTP |
| **Internal Mutation** | A server-only write operation, not exposed via HTTP |
| **Context** | The shared state passed to all procedure handlers |
| **Router** | A collection of procedures grouped under a namespace |

## Requirements

- TypeScript 5.0+
- Node.js 20+

## Installation

```bash
npm install @deessejs/server
```

## Create Your First API

<Steps>
  <Step>
    ### Define Context

    Start by defining your context with the services you need:

    ```typescript
    import { defineContext } from '@deessejs/server'
    import { z } from 'zod'

    const { t, createAPI } = defineContext({
      context: {},
    })
    ```

  </Step>
  <Step>
    ### Define a Query

    Create a typed query that fetches data:

    ```typescript
    const getUser = t.query({
      args: z.object({ id: z.number() }),
      handler: async (ctx, args) => {
        return { id: args.id, name: 'John' }
      },
    })
    ```

  </Step>
  <Step>
    ### Define a Mutation

    Create a mutation that modifies data:

    ```typescript
    const createUser = t.mutation({
      args: z.object({ name: z.string() }),
      handler: async (ctx, args) => {
        return { id: 1, name: args.name }
      },
    })
    ```

  </Step>
  <Step>
    ### Export the API

    Combine everything into a router and export:

    ```typescript
    export const api = createAPI({
      router: t.router({
        users: t.router({
          get: getUser,
          create: createUser,
        }),
      }),
    })
    ```

  </Step>
</Steps>

<Callout type="idea" title="Internal Procedures">
Use `t.internalQuery` and `t.internalMutation` for server-only procedures that should not be exposed via HTTP.
</Callout>

## Setup Server

**Next.js App Router** — `app/api/[...route]/route.ts`

```typescript
import { createServerHandler } from '@deessejs/server-next'
import { api } from '@/api'

export const { api: serverApi, startServer } = createServerHandler({ api })
```

**Hono** — `src/index.ts`

```typescript
import { createHonoHandler } from '@deessejs/server-hono'
import { Hono } from 'hono'
import { api } from './api'

const app = new Hono()
app.route('/api', createHonoHandler({ api }))
```

## Setup Client

**React** — `src/lib/client.ts`

```typescript
import { createClient } from '@deessejs/client-react'
import { httpBatchLink } from '@deessejs/client'

export const client = createClient({
  links: [httpBatchLink({ url: '/api' })],
})

// Call procedures
const user = await client.users.get.query({ id: 1 })
await client.users.create.mutation({ name: 'Jane' })
```

**Vanilla** — `src/client.ts`

```typescript
import { createClient } from '@deessejs/client'

export const client = createClient({ url: '/api' })

// Call procedures
const user = await client.users.get({ id: 1 })
```

<Callout type="warn" title="Security">
Only `query` and `mutation` are exposed via HTTP. Use `internalQuery` and `internalMutation` for server-only operations.
</Callout>

## FAQ

<FaqAccordion items={[
  {
    question: "What's the difference between queries and mutations?",
    answer: "Queries are read operations exposed via HTTP. Mutations are write operations. Internal operations (internalQuery/internalMutation) are server-only."
  },
  {
    question: "How does caching work?",
    answer: "Procedures return cache keys with results. The cache system automatically invalidates keys when mutations are called."

    question: "Is DeesseJS RPC framework-agnostic?",
